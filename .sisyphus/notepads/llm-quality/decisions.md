# Decisions — llm-quality

## 사용자 프로필 (인터뷰 확정)
- 체형 목표: **skinny → 크게.** 두께(등·가슴 density, 로우·데드) + 너비(어깨 측면·랫 V테이퍼). "전체 골고루."
- 훈련 목표: 근비대(hypertrophy). 근력/1RM은 부차.
- 체중 증가 = 성공 지표 중 하나 (말라서 벌크).

## 불만 원인 (확정)
1. 너무 뻔함/일반론 — 개인화 부족.
2. "신규" 도배 — 분석 단위가 종목 이름이라 3회+ 쌓여야 트렌드. 대부분 신규.
3. 실행 지침 약함 — 구체성 부족.
(톤=스파르타는 문제 아님. 유지.)

## 해결 방향 (확정)
- **분석 단위를 부위(bodyPart)로 승격.** 등=랫풀다운+로우+풀업 합산 → 즉시 신호. "신규 도배" 근본 해결.
- **코치/리포트에 rich data 주입**: 부위별 주간 볼륨·빈도·마지막 훈련일, 체중 추세. (현재 coach는 top-set 6개만 봄.)
- **역할 분리**:
  - 코치(매일): 짧고 오늘 중심. 몸 전체 스냅샷 1줄 + 최근 피드백 + 구체 처방 1개. 스타일 A. 스파르타 톤 유지.
  - 리포트(주간): 깊은 분석. 부위별 볼륨 밸런스 / 약점·방치 경고 / 체중·총볼륨 추세. 두께·너비 목표를 렌즈로.
- **체중 추적 신규**: 주 1~2회 가볍게 입력. jsonb 저장(무마이그레이션). 코치·리포트에 전달.
- 체형 목표(두께/너비/크게)는 프로필 필드 대신 **프롬프트에 내장** (단일 오너 앱).

## 스타일 A 채택 예시
"랫풀다운 25→32kg 상승. 최근 5회 중 등4·하체1·가슴1 — 근비대는 균형. 하체가 비었어.
 오늘 하체 가서 스쿼트 반복부터. 등은 32kg 유지 9회."
